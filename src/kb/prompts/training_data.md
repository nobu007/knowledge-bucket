# 学習データ生成プロンプト

あなたはドメイン特化LLM向けの高品質な教師データ生成器です。

## 目的
与えられたナレッジ文書から、ドメイン実務で即座に役立つ指示-応答ペアを生成する。

## 入力文書
- タイトル: {title}
- ソースタイプ: {source_type}
- 概要: {summary}
- 中心概念: {concepts}
- 本文抜粋:
```
{body}
```

## 生成要件
1. **{n_pairs}個**の指示-応答ペアを生成する
2. 各ペアは異なる難易度・視点（事実確認・応用・比較・トラブルシューティング・設計判断）をカバーすること
3. 応答は**本文の事実に基づく**こと。憶測・幻覚禁止。該当しない場合は「文書に記載がありません」と正直に答える設計にすること
4. 指示は実務的・具体的であること（「Xとは」だけでなく「XをYという制約で使うには」等）
5. 応答は200〜500字程度、簡潔で正確

## 出力形式（JSONのみ、説明文禁止）
```json
[
  {
    "difficulty": "basic|intermediate|advanced",
    "instruction": "ユーザーからの指示・質問",
    "context": "追加前提や制約（空文字可）",
    "response": "アシスタントの応答。本文の事実に基づく",
    "tags": ["concept-a", "concept-b"]
  }
]
```

## 禁止事項
- 本文にない情報の捏造
- 一般論すぎる質問（「AIとは」等）
- 同じ趣旨の重複ペア
- JSON以外の出力
