# Web Article Analyzer

## 対象

Web記事、ブログポスト、技術記事、ニュース記事。

## 抽出対象

- タイトル（元記事の<title>または<h1>）
- 本文の主要な内容
- 著者名（わかる場合）
- 公開日（わかる場合）
- URL（sourceとして既に与えられているはず）

## 分析のポイント

1. 記事の核心を200〜400字で要約する
2. なぜこの記事が自分にとって重要かを1〜2文で書く
3. 最大5個の重要ポイントを抽出する
4. 記事の中心的な技術概念や手法をprimary_conceptsに挙げる
5. 関連しそうだが中心ではない概念をcandidate_conceptsに挙げる
6. 言及されているツール、ライブラリ、組織、人物をentitiesに挙げる

## 入力形式

```
Title: <記事タイトル>
Source: <URL>

<記事本文>
```
